MONITORING KENDARAAN

PEMKOT SALATIGA

ESP32 + NEO-6M + SIM800L
Teguh Prasetyo, S.Kom., M.Si.

Tujuan Monitoring

  • Kendaraan pengangkut sampah
  • Ambulan
  • Mobil dinas
  • Dll

Jenis Kendaraan

  • Motor
  • Roda 3
  • Mobil
  • Truk
  • Lainnya

Pembagian Team

  • 1 team terdiri dari 2-4 orang
  • Team bisa fleksibel, boleh pindah team
  • Tentukan team sebelum mulai mengerjakan

Team 1: Hardware Engineer

Firmware & Payload Standardization

Tugasnya adalah memastikan ESP32 di setiap jenis kendaraan mengirim data dalam format yang seragam, meskipun sensornya berbeda.

Tugas Utama:

  • Membuat firmware yang menyertakan Vehicle_ID dan Vehicle_Type (1 untuk Motor, 2 untuk Mobil, 3 untuk Truk) dalam setiap paket data
  • Mengatur interval pengiriman data (misal: Truk setiap 10 detik, Motor setiap 5 detik)
  • Implementasi protokol pengiriman yang efisien (JSON via MQTT)

⚠ Ketergantungan: Team 2 tidak bisa memproses data jika struktur JSON (kontrak data) dari Team 1 belum didefinisikan secara baku.

Sample JSON

{ "v_id": "TRUCK-01", "v_type": "heavy_truck", "lat": -7.3305, "lng": 110.5084, "speed": 60, "timestamp": "2026-04-20T10:00:00Z" }

Team 2: Backend & Database Architect

Multi-Vehicle Handling

Tugasnya adalah membangun "pintu masuk" yang mampu menampung ribuan data dari banyak kendaraan sekaligus.

Tugas Utama:

  • Merancang database (misal: PostgreSQL dengan ekstensi PostGIS, atau MySQL dengan Spatial Data Types) untuk menyimpan koordinat geografis
  • Membuat tabel Vehicles yang menyimpan metadata (plat nomor, jenis kendaraan, kapasitas tangki, dll)
  • Membangun API yang memisahkan data masuk berdasarkan ID kendaraan masing-masing

⚠ Ketergantungan: Team 3 tidak bisa menghitung statistik jika data tidak tersimpan dalam struktur multi-vehicle yang rapi.

Team 3: Logic & Analytics Developer

Engine Rules & Alerts

Tugasnya adalah mengolah data mentah menjadi informasi yang berguna bagi OPD pemilik armada.

Tugas Utama:

  • Geofencing: Membuat fungsi untuk mendeteksi jika kendaraan keluar dari area yang ditentukan (misal: Truk tidak boleh keluar jalur provinsi)
  • Behavior Analytics: Menghitung kecepatan rata-rata untuk mendeteksi overspeeding
  • Maintenance Reminder: Logika untuk memberi peringatan servis berdasarkan total jarak tempuh yang dihitung dari akumulasi koordinat GPS

⚠ Ketergantungan: Team 4 tidak bisa menampilkan peringatan atau grafik performa jika logikanya belum matang di sisi server.

Team 4: Frontend Developer

Management Dashboard

Tugasnya adalah menyajikan data kompleks menjadi tampilan yang mudah dipantau oleh admin atau pemilik perusahaan.

Tugas Utama:

  • Live Map View: Menampilkan banyak marker kendaraan di peta secara bersamaan dengan ikon yang berbeda (ikon motor, mobil, dan truk)
  • Vehicle Switching: Fitur untuk memilih kendaraan tertentu dan melihat riwayat perjalanannya (history playback)
  • Filtering: Membuat filter agar admin bisa melihat "hanya truk yang sedang aktif" atau "motor yang sedang berhenti"

Perangkat Keras (Sisi Kendaraan/Node)

Mikrokontroler ESP32 (Sangat disarankan karena memiliki Dual-Core yang stabil untuk menangani GPS dan pengiriman data Wi-Fi/GPRS secara bersamaan)
Modul GPS NEO-6M atau NEO-7M. Modul ini murah dan akurat untuk mendapatkan koordinat Lat/Lng
Modul Konektivitas SIM800L / SIM900A: Jika kendaraan bergerak jauh (menggunakan kartu SIM untuk data GPRS/2G). ESP32 Built-in Wi-Fi: Jika hanya untuk simulasi di area kantor/rumah yang terjangkau Wi-Fi
Antena Eksternal Sangat disarankan untuk modul GPS agar locking satelit lebih cepat (terutama jika diletakkan di dalam dashboard kendaraan)
Power Supply Buck Converter (LM2596) untuk menurunkan tegangan aki kendaraan (12V) menjadi 5V yang aman bagi ESP32
Penyimpanan Cadangan (Opsional) Micro SD Card Module (untuk menyimpan data jika koneksi internet terputus / offline logging)

Infrastruktur Server & Database

Database MySQL versi 8.0+ (Wajib versi 8 ke atas agar fungsi ST_Distance_Sphere dan indeks spasial berjalan optimal)
MQTT Broker Mosquitto. Ini adalah standar protokol IoT untuk menangani komunikasi ribuan sensor dengan beban memori yang sangat kecil
Application Server Ubuntu Server (bisa diinstal di PC bekas atau layanan Cloud)
Backend Framework Node.js, Python (FastAPI/Flask), atau PHP (Laravel)

Alat Pendukung Pengembangan (Tooling)

Personel 1 (Hardware) Arduino IDE atau VS Code + PlatformIO untuk memprogram ESP32
Personel 2 (Backend) DBeaver atau MySQL Workbench untuk mengelola tabel spasial dan memantau koordinat yang masuk
Personel 3 (Logic) Postman atau Insomnia untuk melakukan testing API sebelum datanya dikonsumsi oleh bagian Frontend
Personel 4 (Frontend) Google Maps API Key atau Mapbox (untuk menampilkan peta digital di dashboard)
