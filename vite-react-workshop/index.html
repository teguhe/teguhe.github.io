Vanilla JS vs Modern Framework

❌ Vanilla JS / HTML / CSS

  • Setiap halaman = file HTML terpisah
  • Pengelolaan DOM manual satu-per-satu
  • Kode JS, CSS, HTML bercampur
  • Sulit dipakai ulang (reusable)
  • Project besar = semakin berantakan
  • Tidak ada sistem component
  • Setup build manual (bundler, minifier)

✅ Modern Framework (React)

  • Single Page Application (SPA) — 1 HTML, banyak "halaman"
  • DOM manipulation declarative
  • Kode terstruktur dalam Component
  • Component bisa dipakai ulang di mana saja
  • Arsitektur modular, mudah dipelihara
  • Virtual DOM = rendering lebih cepat
  • Ekosistem build tools (Vite) otomatis

Apa itu Single Page Application (SPA)?

📄
Traditional Web
Setiap klik link = browser minta HTML baru ke server → server kirim HTML → browser reload seluruh halaman. Setiap reload ada jeda loading yang terasa "patah".
⚡
Single Page Application
Cukup satu HTML. Semua "halaman" sudah ada di browser. Ketika klik link, JavaScript menukar konten secara cepat tanpa reload. Terasa seperti app native.

Contoh SPA yang Pernah Kamu Pakai:

📱 Gmail📘 Facebook🐦 Twitter/X📊 Trello🗂️ Notion🎵 Spotify Web

Cara Kerja SPA vs Traditional

🔁 Traditional Multi-Page

  • User klik link
  • Browser kirim request ke server
  • Server render HTML baru
  • HTML dikirim balik ke browser
  • Browser unload halaman lama
  • Browser load halaman baru
  • ⏱️ Loading setiap kali

⚡ Single Page Application

  • User klik link / button
  • JavaScript intercept request
  • Component React "merender" konten baru
  • Hanya bagian yang berubah di-update
  • Tidak ada unload/load halaman
  • Transisi terasa INSTAN
  • 🚀 Sekali load, semua cepat

💡 React menggunakan Virtual DOM — representasi ringan dari DOM asli. Ketika state berubah, React membandingkan Virtual DOM lama dan baru, lalu cuma meng-update bagian yang berbeda di DOM asli.

Kenalan Dulu: Node.js & NPM

🟢
Node.js
Runtime JavaScript di luar browser. Permite kamu menjalankan JavaScript di server/terminal. Dibutuhkan untuk menjalankan build tools modern seperti Vite, Webpack, dll.
📦
NPM (Node Package Manager)
Toko online untuk library JavaScript. Sama kayak Play Store tapi untuk kode. Ada jutaan package siap pakai — React, Tailwind, Express, dll.
# Cek versi Node.js dan NPM $ node --version $ npm --version # Buat proyek React baru dengan Vite $ npm create vite@latest my-app -- --template react $ cd my-app $ npm install # Jalanin development server $ npm run dev

Struktur Folder React

📂 Struktur Folder

my-app/ ├── node_modules/ # Library ter-install ├── public/ # File statis publik ├── src/ # Kode sumber utama │ ├── assets/ │ ├── App.jsx # Component utama │ ├── main.jsx # Entry point │ └── index.css ├── index.html # HTML root ├── package.json # Project config └── vite.config.js

Penjelasan Singkat

  • src/ — semua kode React di sini
  • App.jsx — component utama
  • main.jsx — mount React ke DOM
  • index.html — HTML yang jadi "rumah" React
  • public/ — file statis (favicon, gambar)
  • node_modules/ — jangan disentuh 😄

JSX — HTML di Dalam JavaScript

HTML Biasa

<!-- File terpisah, tidak bisa pakai variabel JS --> <div class="card"> <h2>Nama Produk</h2> <p>Harga: Rp100.000</p> </div>

JSX (React)

// Bisa pakai variabel JS langsung! function Card() { const name = "Laptop"; const price = 15000000; return ( <div className="card"> <h2>{name}</h2> <p>Harga: Rp{price.toLocaleString()}</p> </div> ); }

Aturan Penting JSX:

  • ✅ Pakai className bukan class
  • ✅ Tutup semua tag (<br />, <img />)
  • ✅ Curly braces {'{ }'} untuk masukkan variabel JS
  • ✅ Satu root element per component (bungkus dengan <></> atau <div>)

Vite & React

React Hooks — useState

Apa itu Hooks?

Hooks adalah fungsi khusus React yang memungkinkan kamu menggunakan fitur React (state, lifecycle, dll) di dalam functional component. useState adalah hook paling dasar — digunakan untuk menyimpan dan mengupdate data yang berubah.
// import useState dari React import { useState } from 'react'; function Counter() { // count = state saat ini, setCount = fungsi untuk update state const [count, setCount] = useState(0); return ( <div> <p>Angka: {count}</p> <button onClick={() => setCount(count + 1)}> Tambah </button> </div> ); }

💡 Konsep penting: Setiap kali state berubah, React Otomatis merender ulang component dan memperbarui DOM. Kamu nggak perlu panggil document.getElementById() lagi!

Tailwind CSS

Utility-First CSS

❌ CSS Tradisional (file .css)

  • Buat class di file .css terpisah
  • Nama class harus dipikirin sendiri
  • Susah track: class ini di file mana?
  • Sering ada conflict naming
  • Loading CSS terpisah = extra request
  • Sulit bikin responsif

✅ Tailwind CSS (Utility-First)

  • Class udah tersedia langsung di HTML
  • Nama class = nama stylenya (self-documenting)
  • Tinggal baca class di HTML, langsung paham
  • Tidak ada conflict naming
  • Purge otomatis — hanya CSS yang dipakai yang di-load
  • Bikin responsif sangat mudah
<!-- Tailwind: semua styling langsung di class! --> <div className="flex items-center justify-between p-4 bg-white rounded-lg shadow-md"> <h1 className="text-2xl font-bold text-gray-900">Judul</h1> <button className="px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600"> Klik Saya </button> </div>

Layout Responsif: Flexbox & Grid

↔️
Flexbox (flex)
Atur layout dalam satu arah — horizontal atau vertical. Cocok untuk navbar, card row, button group.
⬛
Grid (grid)
Atur layout dalam dua arah — baris dan kolom sekaligus. Cocok untuk gallery, dashboard, card grid.
📱
Responsif (sm:, md:, lg:)
Prefix breakpoint untuk styling berbeda di tiap ukuran layar.
🎯
Spacing & Colors
Padding: p-4, Margin: mx-auto, Colors: bg-blue-500, text-white.
